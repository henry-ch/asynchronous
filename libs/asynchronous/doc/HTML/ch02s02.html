Shutting down

Shutting down a thread turns out to be harder in practice than expected, as shown by several posts of surprise when Boost.Thread tried to match the C++ Standard. Asynchronous hides all these ugly details. What users see is a proxy object, which can be shared by any number of objects executing within any number of threads.

When the last instance of the inner-ActiveComponent scheduler object is destroyed, the scheduler thread is stopped. When the last instance of a scheduler proxy is destroyed, the scheduler thread is joined. It's as simple as that. This makes threads shared objects.
